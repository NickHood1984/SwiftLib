# SwiftLib 1.5.0

发布日期：2026-06-06

## 重点更新

- **引用输出更稳**：新增统一 CSL 导出服务，App、CLI、Word/WPS 插件和 DOCX 刷新现在共用同一套 Reference -> CSL 转换逻辑，减少不同入口的输出漂移。
- **Word 插件插入前预检**：插入引用前会检查关键字段、渲染结果和异常片段；缺少会导致坏引文的字段时会阻止插入并提示原因。
- **文献库审计与修复 CLI**：新增 `audit-library` 和 `repair-library`，可检查 DOI 前缀、稳定期刊访问日期、作者字段异常、拼音姓/名错位、重复作者序列、期刊证据类型错配和疑似重复译本。
- **中文元数据路线更保守**：中文期刊刷新和按标识符导入会跳过不合适的 CrossRef 路线，降低中文标题、作者顺序和卷期页码被拉丁化或覆盖的风险。

## 改进

- DOI、作者/编者/译者、语言标签、访问日期和 `other` 类型期刊证据会在入库与引用导出前统一规范化。
- CrossRef 解析会使用 `sequence=first` 锚定第一作者，并更保守地修正中文拼音作者姓/名错位，避免误伤西文作者。
- 内置 GB/T 7714 数字样式支持短 ID `gb-t-7714-2015-numeric`，DOI-only 期刊记录不再被误标为在线文献。
- DOCX 审计新增正文可见引用编号与参考文献编号对照，能发现正文和参考文献表不一致的问题。
- 站点适配器配置升级到 v3，补充 CNKI 图书、万方、维普和 Highwire 类期刊页面，并在网页提取前自动检查远程配置。
- Word 插件本地服务限制到 loopback，并改进 HTTP 分片请求读取、孤儿引用占位和 typed citation-item 合约。
