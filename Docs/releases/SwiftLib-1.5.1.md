# SwiftLib 1.5.1

发布日期：2026-06-11

## 重点更新

- **元数据刷新不再永久挂起**：CNKI/百度学术人机验证等待现在响应任务取消，刷新超时后正常返回；百度学术验证面板补上了主界面的呈现链路；离屏 HTML 解析增加 15 秒加载超时。进度条上的取消按钮对单条刷新同样生效。
- **批量刷新更快更稳**：单源软超时（15 秒）让 CrossRef/OpenAlex/Semantic Scholar 并行抓取不再等最慢的源；修复 `HostRateLimiter` 并发下完全失效的 actor 重入问题与 `HostCircuitBreaker` 半开逻辑漏洞，批量操作不再成批 429、触发熔断。
- **中文候选评分更诚实**：万方/维普候选改用与 CNKI 同量纲的多字段评分（题名/作者/年份/期刊），移除 45% 人为下限；中文作者名走 Han 感知解析，不再被拆成"姓=三 名=张"；重新接回 v1.4.0 丢失的百度学术兜底检索链路。
- **CSL 样式导入修复**：样式 `<id>` 为 URL 时导入不再失败（文件导入、Word 插件上传、XML 导入三条路径全部修复）；重新导入或删除样式后渲染立即生效；引用预览渲染改用 citeproc-js 快速状态重置，大幅降低每条渲染延迟。

## 改进

- 标题检索发现的 DOI 经 CrossRef 反向验证，明显不符时丢弃并从各源结果中清除，防止错误 DOI 覆盖正确的标题/作者。
- 中文浏览器检索通道（万方/维普）在批量并发占用时排队等待而非静默返回"无结果"。
- 修复 `ChineseMetadataConsensus` 作者选取在全无中文作者名时被最低优先级源覆盖的缺陷。
- 统一浮层视觉风格（进度提示条、CSL 消息条、队列通知共用同一套设计令牌）；待确认元数据窗口改为隐藏式标题栏；候选卡片信息精简并全部中文化。
- 为 `HostRateLimiter` 与 `HostCircuitBreaker` 补齐单元测试。
